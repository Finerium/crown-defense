LOCKED INVARIANTS (architecture, present from the first line of code; verified, never bolted on):
- **Fail-safe.** On failure of the reasoning layer (the LLM) or loss of the control plane, the system degrades toward monitor and alert and maintains existing containment, but NEVER initiates new destructive action (deny-by-default).
- **Autonomy dial.** A four-position runtime dial (monitor-only, alert-and-recommend, human-gated, full-auto), default monitor-only; destructive actions are classified Auto, Ask-to-Act, or Never-Auto; human-gated destructive actions require dual control (a second distinct approver) and are time-boxed with auto-rollback and one-click revert. - **Audit immutability.** Every autonomous action and every human approval is recorded in an append-only, hash-chained, tamper-evident, attributable, exportable audit log; the audit substrate exists before the first action is possible; 24-hour reconstruction is supported.
- **Detection robustness.** Multi-signal fusion; a destructive verdict requires at least two corroborating signals unless the canary fast-path fired; a file-format-validation signal counters intermittent encryption (entropy alone is insufficient and is evadable).

- **Sovereignty.** The production LLM is self-hosted on-premise; security telemetry never egresses to a third-party cloud. A cloud LLM API is permitted ONLY for development and testing of Crown Defense, never as the production answer.
- **Self-protection.** A mutual-TLS authenticated and encrypted agent-to-control-plane channel; kernel-mode anti-tamper for the agent; a hardened, deny-by-default control plane; signed, staged, fail-safe agent updates.
